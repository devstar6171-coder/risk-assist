# Team Oscar: AI-Driven Defence Risk Management
**Target State:** An end-to-end web application that automates document ingestion, risk analysis, and compliance reporting using Vertex AI.


## Project Technical Phases


### Phase 1: Environment & Security Foundation (IN PROGRESS)
*  [x] **Service Enablement:** Enable Vertex AI, Cloud Run, Document AI, and Sensitive Data Protection (DLP) APIs.
*  [x] **Secure Ingestion Storage:** Create a GCS bucket (`team-oscar-ingestion-[PROJECT_ID]`) with Public Access Prevention enforced.
*  [ ] **Service Account (SA):** Create `team-oscar-sa` with `roles/aiplatform.user`, `roles/storage.objectAdmin`, and `roles/dlp.user`.
*  [ ] **DLP Inspection Template:** Define a custom template to identify and mask UK-specific PII (National Insurance numbers, Passport numbers) and generic PII.


### Phase 2: Web Application & Input Portal (UIP)
*  [ ] **Frontend Build:** Develop a React/Next.js or Streamlit interface for file uploads and real-time status tracking.
*  [ ] **API Backend:** Develop a FastAPI/Python backend to orchestrate the workflow.
*  [ ] **Document Normalization:** Configure a Document AI 'OCR Processor' to convert unstructured uploads into standardized text.
*  [ ] **Sanitization Pipeline:** Integrate the DLP API to strip sensitive data from normalized text before it hits the AI engine.
*  [ ] **Containerization:** Create a `Dockerfile` and deploy the unified app to **Cloud Run**.


### Phase 3: Frontier AI Risk Engine (FAIRE)
*  [ ] **Gemini 1.5 Pro Integration:** Implement the core reasoning engine using the Vertex AI SDK.
*  [ ] **Prompt Engineering:** Define a system instruction set that forces the LLM to categorize risks into: Kinetic Safety, Cybersecurity, and OPSEC.
*  [ ] **Mathematical Scoring:** Ensure the LLM returns a JSON object containing scores (1-10) for Mitigation Cost, Technical Complexity, and Time-to-Deployment.
*  [ ] **RAG Implementation:** Set up Vertex AI Vector Search to ground the engine in actual Defence taxonomies.


### Phase 4: Structured Outcome & Dashboard (SOIB)
*  [ ] **Risk Visualizer:** Build an interactive dashboard within the web app to triage and archive identified risks.
*  [ ] **Export Logic:** Deploy a Cloud Function to generate a signed PDF/JSON compliance artifact once a risk profile is "Baselined" by the user.
